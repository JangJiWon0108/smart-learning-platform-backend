# 실행 가이드 (Execution Guide)

프로젝트 백엔드 및 프론트엔드 로컬 실행 방법.

---

## 1. 백엔드 (Backend)

### 서버 실행 (main FastAPI)
- 프론트엔드 연동을 위한 메인 API 서버 구동.
```bash
cd smart-learning-platform-backend
uv run uvicorn api.app:app --reload --app-dir .
```

### MCP 서버 실행 (추천 검색 Tool)
- Vertex AI Search MCP는 **streamable-http**로만 붙습니다. 추천 에이전트·A2A 실행 전에 **8200 포트**로 띄웁니다.
```bash
cd smart-learning-platform-backend
uv run python -m mcp_server.vertexai_search.server --transport streamable-http --host 127.0.0.1 --port 8200
```
- 엔드포인트는 기본 `http://127.0.0.1:8200/mcp`이며, 변경 시 `.env`의 `VERTEXAI_SEARCH_MCP_URL`을 맞춥니다.
- (선택) stdio로만 도구를 손보고 싶을 때: `uv run python -m mcp_server.vertexai_search.server`

### A2A route agent 실행 (solver/recommendation/visualization/fallback)
- 메인 API(`/chat/stream`)가 내부적으로 각 route A2A 서비스의 `/stream`을 호출합니다.
- 각 서비스는 별도 포트로 실행합니다.
```bash
cd smart-learning-platform-backend

# solver (8101)
uv run uvicorn a2a_remote_routes.services:solver_app --reload --app-dir . --host 127.0.0.1 --port 8101

# recommendation (8102)
uv run uvicorn a2a_remote_routes.services:recommendation_app --reload --app-dir . --host 127.0.0.1 --port 8102

# visualization (8103)
uv run uvicorn a2a_remote_routes.services:visualization_app --reload --app-dir . --host 127.0.0.1 --port 8103

# fallback (8104)
uv run uvicorn a2a_remote_routes.services:fallback_app --reload --app-dir . --host 127.0.0.1 --port 8104
```

### CLI 테스트
- 개별 쿼리에 대한 에이전트 동작 단독 확인.
```bash
uv run python main.py "포인터 관련 문제 추천해줘"
```

---

## 2. 프론트엔드 (Frontend)

### 초기 설정
- 환경 변수 설정 및 의존성 모듈 설치.
```bash
cd smart-learning-platform-frontend
cp .env.example .env
npm install
```

### 개발 서버 실행
- UI 확인을 위한 Vite 개발 서버 구동.
```bash
npm run dev
```
