# 백엔드 개발 표준 (Backend Development Standards)

`smart-learning-platform-backend` 프로젝트의 코드 일관성과 유지보수성 향상을 위한 가이드라인.

---

## 1. 파일 명명 규칙
- **LLM 에이전트**: `llm_agents/{route}/{역할}_agent.py` (예: `llm_agents/solver/solver_agent.py`)
- **전처리 노드**: `nodes/{route}/{역할}_nodes.py` (예: `nodes/solver/solver_nodes.py`)
- **공통 처리**: 모든 라우트가 공유하면 `common/` 하위에 둔다. 예: `llm_agents/common/query_rewrite_agent.py`, `nodes/common/router.py`
- **라우트 폴더**: `solver/`, `recommendation/`, `visualization/`, `fallback/`을 사용한다. 단, `fallback`은 LLM Agent만 두고 Function Node가 없으면 `nodes/fallback/`은 만들지 않는다.
- **스키마**: `{역할}_output.py` (예: `solver_output.py`)
- **설정 파일**: `properties.py`
- **임포트 순서**: 1. `__future__` 2. 표준 라이브러리 3. 서드파티 4. 로컬 모듈(절대경로)
- **접근 제한**: 공개 API(접두사 없음), 내부 헬퍼 함수(`_`), 모듈 상수(`_UPPER_SNAKE_CASE`)

## 2. 주석 및 독스트링 (Docstring)

### 2.1 파일 및 모듈
- 모든 `.py` 파일 최상단에 해당 모듈의 역할과 핵심 로직을 설명하는 한글 독스트링 작성 필수.

### 2.2 함수 및 클래스 (Google 스타일)
- 모든 인자와 반환값에 대한 설명 기재.
- 단순 내부 헬퍼가 함수명만으로 의미가 명확하면 한 줄 docstring을 허용한다. 단, 공개 함수와 콜백, 외부 API/MCP/A2A 경계 함수는 Args/Returns를 작성한다.
- ADK 노드 함수의 경우 `state에 저장되는 값:` 항목 명시 필수.
```python
def node_func(param1: str) -> None:
    """
    {요약 및 역할 설명}
    Args: param1 설명
    Yields: Event
    state에 저장되는 값:
        - key_name: 설명
    """
```

### 2.3 논리적 블록 구분 및 인라인 주석
- 연관된 코드 뭉치(설정, 헬퍼, 실행부 등)는 시각적 구분선(`# ─── 파트명 ───`)으로 분리.
- 주요 모듈 변수 위에 목적 기재 및 복잡한 로직의 단계별 주석(`# 1단계: ...`) 추가.
- `llm_agents/{route}/`의 에이전트 파일은 섹션명을 아래 3개로 통일.
  1. `# ─── 모듈 임포트 ───`
  2. `# ─── 설정 로드 ───`
  3. `# ─── 에이전트 정의 ───`
- `llm_agents/{route}/`에서는 `# 설정 로드`, `# 환경 설정 및 전역 상수`, `# Solver 에이전트 구성`처럼 파일별로 다른 섹션명을 사용하지 않는다.
- `nodes/{route}/`의 함수형 노드 파일은 필요한 섹션만 아래 이름으로 통일.
  1. `# ─── 모듈 임포트 ───`
  2. `# ─── 설정 로드 ───`
  3. `# ─── 상수 정의 ───`
  4. `# ─── 헬퍼 함수 ───`
  5. `# ─── 노드 함수 ───`
- `nodes/{route}/`에서는 `# 임포트`, `# 설정 및 상수`, `# 쿼리 전처리 함수`, `# 의도 기반 라우팅 함수`처럼 파일별로 다른 섹션명을 사용하지 않는다.
- `schemas/`의 Pydantic 스키마 파일은 섹션명을 아래 2개로 통일.
  1. `# ─── 모듈 임포트 ───`
  2. `# ─── 스키마 정의 ───`
- `callbacks/`의 ADK 콜백 파일은 필요한 섹션만 아래 이름으로 통일.
  1. `# ─── 모듈 임포트 ───`
  2. `# ─── 상수 정의 ───`
  3. `# ─── 헬퍼 함수 ───`
  4. `# ─── 콜백 함수 ───`
- `callbacks/`에서는 `# 임포트`, `# 설정 및 상수`, `# 내부 헬퍼 함수`처럼 파일별로 다른 섹션명을 사용하지 않는다.
- **한글 단일행 주석(`# ...`) 종결**: 서술어 어미(예: ~한다, ~합니다, ~됩니다, ~있습니다, ~송신합니다) 대신 명사형·명사구로 끝낸다. 목적이면 `… 목적`, `… 처리`, 조건이면 `… 조건`, 배경이면 `… 가능`처럼 짧게 쓴다. 구분선(`# ─── … ───`), 영문-only 주석, 인라인 끝 주석은 예외로 둘 수 있다.

### 2.4 코드 가독성 원칙
- 주석은 코드가 이미 말하는 “무엇을 하는지”보다 배경, 제약, 예외 처리 이유를 설명한다.
- 상수명과 함수명으로 의도가 드러나면 설명 주석을 추가하지 않는다.
- 헬퍼 함수는 중복 제거, 외부 경계 정리, 복잡도 분리에 실제 도움이 될 때만 만든다.
- 긴 인라인 문자열(LLM instruction, query, prompt)은 `_INSTRUCTION` 같은 모듈 상수로 분리한다.
- 외부 연동(MCP, A2A, Vertex AI)은 “연결 설정”, “요청 구성”, “응답 변환” 책임을 분리한다.
- 실패 처리에서는 `print`/`traceback.print_exc()` 대신 logger를 사용하고, 사용자 응답 메시지는 별도 헬퍼로 정리한다.
- 표준을 맞추기 위한 장황한 독스트링은 피한다. 짧은 함수는 한 문장으로 충분하며, 공개 경계 함수만 상세히 쓴다.

## 3. 타입 시스템 및 스키마

### 3.1 타입 힌트
- 파라미터 및 반환값 타입 명시 필수 (Python 3.10+ `X | None` 문법 권장).

### 3.2 Pydantic 스키마 (`schemas/`)
- 필수 필드는 `Field(...)`, 선택 필드는 `Field(default=...)` 사용.
- 모든 필드에 `description` 작성 및 선택지 제한 시 `Literal` 활용.
- 스키마 파일은 `# ─── 모듈 임포트 ───`와 `# ─── 스키마 정의 ───` 섹션을 사용.
- 클래스 docstring의 `Attributes:` 항목은 `name: 설명` 형식으로 작성하고, 정렬용 공백을 넣지 않는다.
```python
# ─── 모듈 임포트 ───────────────────────────────────────────────────────────
from pydantic import BaseModel, Field


# ─── 스키마 정의 ───────────────────────────────────────────────────────────
class SampleOutput(BaseModel):
    """구조화 출력 설명."""

    value: str = Field(..., description="값 설명")
```

## 4. Google ADK 구현 가이드

프로젝트 핵심 프레임워크인 ADK 활용 시의 반복 패턴 준수.

### 4.1 에이전트 정의 (`llm_agents/{route}/`)
```python
# ─── 모듈 임포트 ───────────────────────────────────────────────────────────
from google.adk import Agent

from config.gemini_retry import GEMINI_GENERATE_CONTENT_RETRY_CONFIG
from config.properties import Settings

# ─── 설정 로드 ─────────────────────────────────────────────────────────────
settings = Settings()

# ─── 에이전트 정의 ─────────────────────────────────────────────────────────
agent = Agent(
    name="intent_agent",  
    model=settings.GEMINI_MODEL_TYPE_INTENT, # 용도별 모델 설정 변수 참조
    generate_content_config=GEMINI_GENERATE_CONTENT_RETRY_CONFIG, # 공통 API 설정 재사용
    output_schema=IntentOutput, # Pydantic 모델 지정
    output_key="intent_result", # state 저장 키 이름
    after_agent_callback=intent_callback_func, # 실행 후 콜백
    tools=[], # 필요 시 도구 목록
    description="사용자 질문 의도 분류 에이전트",
    instruction=_INSTRUCTION # 모듈 상단 상단 정의 지시문
)
```

### 4.2 주요 파라미터 규칙
- **name**: 영문 소문자와 언더스코어 조합의 고유 식별자 사용.
- **model**: `settings` 상수를 통한 모델 지정 (직접 문자열 입력 금지).
- **generate_content_config**: 온도, 재시도 등 공통 설정 파일 참조.
- **output_schema**: LLM 응답 구조화 스키마. 모든 `Field`에 구체적인 `description` 작성 필수.
- **output_key**: 워크플로우 `state`에 결과가 저장될 키 이름 정의.
- **after_agent_callback**: 데이터 재가공 및 상태 조합을 위한 콜백 패턴 활용. `callback_context.state`를 통한 상태 접근 및 가공 후 `state["key"]` 업데이트.
- **tools**: 검색, 외부 API 연동 등 모델이 직접 호출할 함수 목록. MCP 서버 연동은 `McpToolset`을 사용한다.
- **description**: 에이전트 역할 요약 (로깅 및 관리 목적).
- **instruction**: 에이전트 페르소나 및 동작 규칙 정의. 모듈 최상단 `_INSTRUCTION` 상수로 분리 관리.
    1. 역할 선언 (`당신은 ... 입니다`)
    2. state 변수 참조 (`{state_key}` 형식)
    3. 규칙 섹션 구분 (마크다운 헤딩 활용)
    4. 출력 형식 강제 (JSON 응답 시 마크다운 코드 블록 제외 지시)

### 4.3 MCP Toolset 사용
- Vertex 검색 MCP는 **streamable-http**로 띄우고 `StreamableHTTPConnectionParams(url=settings.VERTEXAI_SEARCH_MCP_URL, ...)`로 연결한다.
- 한 `McpToolset`에서 필요한 tool만 `tool_filter`로 노출한다(예: 추천 검색 단계는 `search_exam_questions`만).
- MCP tool 응답을 후속 노드 state로 넘겨야 하면 `after_tool_callback`에서 `context.state`에 저장한다.
```python
from google.adk.tools.mcp_tool.mcp_session_manager import StreamableHTTPConnectionParams
from google.adk.tools.mcp_tool.mcp_toolset import McpToolset

agent = Agent(
    name="vertex_search_agent",
    tools=[
        McpToolset(
            connection_params=StreamableHTTPConnectionParams(
                url=settings.VERTEXAI_SEARCH_MCP_URL,
                timeout=10.0,
            ),
            tool_filter=["search_exam_questions"],
        )
    ],
)
```

### 4.4 노드 및 라우팅 (`nodes/{route}/`)
- LLM 호출 없는 데이터 처리 및 흐름 분기 로직.
- 파라미터 이름과 ADK state 키 이름을 일치시켜 자동 주입 활용.
- 결과값은 `yield Event(state={"key": value})` 형태로 반환하여 글로벌 상태 업데이트.
- 노드 함수는 `Generator[Event, None, None]` 반환 타입을 명시.
- `state에 저장되는 값:` 항목은 `- key_name: 설명` 형식으로 작성하고, 불필요한 정렬용 공백을 넣지 않는다.
```python
# ─── 모듈 임포트 ───────────────────────────────────────────────────────────
from collections.abc import Generator

from google.adk import Event


# ─── 노드 함수 ─────────────────────────────────────────────────────────────
def sample_node(state_key: str) -> Generator[Event, None, None]:
    """
    노드 역할 요약.

    Args:
        state_key: ADK state에서 자동 주입되는 값

    Yields:
        state 업데이트 이벤트

    state에 저장되는 값:
        - next_key: 다음 노드에서 사용할 값
    """
    yield Event(state={"next_key": state_key})
```

### 4.5 콜백 (`callbacks/`)
- ADK 콜백 함수는 `CallbackContext`를 인자로 받고 `types.Content | None` 반환 타입을 명시.
- 콜백이 state를 직접 수정하는 경우 함수 docstring 또는 모듈 docstring에 저장되는 state 키를 명시.
- 내부 변환 로직은 `_helper_name` 형식의 헬퍼 함수로 분리하고 `# ─── 헬퍼 함수 ───` 섹션 아래 배치.
```python
# ─── 모듈 임포트 ───────────────────────────────────────────────────────────
from google.adk.agents.callback_context import CallbackContext
from google.genai import types


# ─── 콜백 함수 ─────────────────────────────────────────────────────────────
def sample_callback(callback_context: CallbackContext) -> types.Content | None:
    """에이전트 실행 후 state를 후처리합니다."""
    callback_context.state["next_key"] = "value"
    return None
```

## 5. 기타 컨벤션

- **에러 핸들링**: API 레이어에서 `HTTPException` 및 한글 메시지 사용. 입력값 방어적 검사 필수.
- **로깅**: `get_logger` 활용. `info`, `warning` 등 적절한 레벨로 기록.
- **비동기 처리**: SSE 스트리밍 시 `AsyncGenerator` 및 `async for` 패턴 준수.

## 6. PR 전 자가 체크리스트
- [ ] 모듈/클래스/함수 한글 독스트링 및 주요 변수 주석 확인
- [ ] 시각적 구분선(`# ───`)을 통한 코드 블록 구조화 여부
- [ ] 타입 힌트 및 Pydantic `description` 누락 확인
- [ ] ADK 지시문 규칙 및 노드 `state` 반환 규칙 준수 여부
- [ ] 임포트 순서(표준/서드파티/로컬) 및 내부 헬퍼 접두사(`_`) 확인
- [ ] 주석/독스트링이 과하게 장황하지 않고, 코드 의도와 제약을 이해하는 데 도움이 되는지 확인
- [ ] 외부 연동 코드는 설정, 호출, 응답 변환 책임이 섞이지 않았는지 확인
- [ ] 한글 `#` 단일행 주석이 명사형 종결 규칙을 따르는지 확인
