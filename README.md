# 정보처리기사 실기 학습을 위한 지능형 플랫폼

정보처리기사 실기 학습을 위한 지능형 플랫폼. 
[Google ADK 2.0](https://google.github.io/adk-docs/) `Workflow`로 의도 분류 후 **문제 해설**·**문제 추천**·**문제 시각화(트레이스)** 세 갈래로 분기합니다.

## 동기

- **주제별로 문제를 찾기 어려웠음**  
  실기를 준비하면서 C 언어 이중 포인터, 자바 업캐스팅처럼 **특정 개념에 맞는 문제만** 골라 연습하고 싶었는데, 기출·복원 자료는 대체로 “N년도 N회차”처럼 시험 단위로만 묶여 있어 원하는 주제를 직접 찾기가 번거로웠음 (관련 문항을 끌어오도록 **RAG(벡터 검색)** 적용할 예정)

- **복잡한 코드 문제는 따라가기 한계가 있었음**  
  자바의 생성자·상속·업캐스팅이나 C의 포인터·재귀처럼 **분기와 메모리 흐름이 길게 이어지는 코드**는 텍스트만으로는 실행 순서를 재현하기 어려웠음. 그래서 **한 줄·한 분기씩 디버깅하듯 설명·추적해 주는 시각화(Tracer) 기능**이 있으면 이해에 도움이 될 것 같음


## 아키텍처 (워크플로)

전체 흐름만 보면 아래와 같습니다. (세부 노드·그래프는 `agent.py` 주석 참고)

```text
                         ┌─────────────┐
                         │  질의 저장   │
                         └──────┬──────┘
                                │
                                ▼
                 ┌──────────────────────────────┐
                 │ 의도 분류                     │
                 └──────────────┬───────────────┘
                                │
                                ▼
                         ┌─────────────┐
                         │   라우터    │
                         └──────┬──────┘
                                │
         ┌──────────────────────┼──────────────────────┐
         ▼                      ▼                      ▼
   ┌───────────────┐    ┌───────────────┐    ┌───────────────────┐
   │ 문제 해설     │    │ 문제 추천     │    │ 문제 시각화       │
   │ RAG 없음      │    │ RAG 기반      │    │ · 트레이스        │
   │ 웹 검색 위주  │    │ 벡터 검색     │    │ · 실행 흐름 추적  │
   │ (예정 수준)   │    │               │    │ · 언어 감지       │
   └───────────────┘    └───────────────┘    └───────────────────┘
```

구현 매핑(코드·`agent.py`): 문제 해설 → `solver` / `solver_agent`, 문제 추천 → `recommendation` / `curator_agent`, 문제 시각화·트레이스 → `visualization` / `tracer_agent`. 그 외 의도는 `other` 폴백.

| 갈래 | 역할 | 에이전트·도구 |
|------|------|----------------|
| **문제 해설** | 풀이·해설 중심 (로컬 문서 RAG 없음) | 전처리 → `solver_agent` · Google Search 위주(예정) |
| **문제 추천** | 주제·약점에 맞는 문항 끌어오기 | 쿼리 임베딩·약점 분석 병합 → `curator_agent` · 벡터 검색(RAG) |
| **문제 시각화·트레이스** | 코드 실행·분기 따라가기 | 언어 감지 → `tracer_agent` |

## 데이터

**2020년~2025년** 정보처리기사 **실기** 기출·복원 문제를 바탕으로 학습·검색 데이터를 구성합니다. (벡터 스토어·목 데이터 등은 해당 연도 범위의 문항을 중심으로 사용)

## 임베딩 전략

벡터 스토어(`data/vector_store.jsonl` 등)에서는 **시험 1문항 = 레코드 1건**으로 두고, 검색에 쓰는 텍스트 필드 `content`는 아래처럼 **구역 레이블**로 묶습니다.

```text
[문제] …문항 본문…
[정답] …모범 답안…
[해설] …해설·근거…
```

## 벡터 스토어 재생성(임베딩)

로컬 `sentence-transformers` 모델로 기출을 임베딩해 `data/vector_store.jsonl`을 재생성할 수 있습니다.

```bash
python -m embedding.build_vector_store
```

모델을 별도 경로(예: `/Users/n-jwjang/jjw/embedding`)에 두었다면 아래처럼 지정합니다.

```bash
EMBEDDING_MODEL_PATH="/Users/n-jwjang/jjw/embedding" python -m embedding.build_vector_store
```
